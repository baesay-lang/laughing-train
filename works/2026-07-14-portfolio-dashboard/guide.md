# 업무 포트폴리오 대시보드 — 사용 가이드

Claude와 함께 만든 결과물을 한곳에 모아 보는 개인 아카이브입니다.

- **대시보드 주소:** https://baesay-lang.github.io/laughing-train/ (북마크해두세요)
- **저장소:** https://github.com/baesay-lang/laughing-train

## 새 결과물을 추가하는 법

가장 쉬운 방법은 **Claude에게 말로 시키는 것**입니다. 어떤 Claude 세션에서든 작업을 끝낸 뒤 이렇게 말하면 됩니다:

> "이 결과물, 내 포트폴리오(laughing-train)에 저장해줘"

그러면 Claude가 저장소의 `CLAUDE.md` 규칙에 따라:

1. 최종 결과물만 골라서 (중간 산출물 제외)
2. `works/날짜-제목/` 폴더에 넣고
3. 제목·카테고리·프로젝트를 자동 분류해 `meta.json`을 만들고
4. 푸시하면 대시보드가 **자동으로 갱신**됩니다 (1~2분 소요)

직접 파일을 넣어도 됩니다. `works/YYYY-MM-DD-제목/` 폴더에 파일을 넣고 푸시하면 끝. `meta.json`이 없으면 폴더명으로 날짜·제목을 추론해 "미분류"로 표시됩니다.

## 대시보드에서 할 수 있는 것

- **타임라인:** 최신순, 월별로 결과물이 정리됩니다
- **카테고리 / 프로젝트 필터:** 왼쪽 칩을 클릭해 좁혀보기
- **검색:** 제목·설명·태그로 즉시 검색
- **상세 보기:** 카드를 클릭하면 파일 목록이 열리고, 문서(md)는 바로 읽기 화면으로, HTML/이미지는 브라우저에서 바로 열립니다
- **이전 버전:** 갱신된 결과물의 옛 버전도 상세 화면에서 열람 가능, GitHub 변경 이력 링크 제공

## 버전 관리

같은 결과물을 업데이트할 때 Claude가 이전 파일을 `archive/v1-날짜/` 폴더로 옮기고 새 버전을 넣습니다. 대시보드 상세 화면에 "이전 버전"으로 자동 표시됩니다.

## 주의

이 저장소와 대시보드는 **공개**입니다. 민감한 정보(개인정보, 사내 기밀)가 담긴 결과물은 올리기 전에 한 번 더 생각해주세요. Claude도 저장 전에 민감해 보이는 내용이 있으면 확인을 요청하도록 설정되어 있습니다.
